## Разработка библиотеки управления PTY-процессами prux и интерфейса ProcessSession на языке Rust

## 1. Введение и обоснование
Целью настоящего проекта является создание надежной, устойчивой к сбоям системы управления процессами в псевдотерминалах (PTY) на языке Rust.
Существующие высокоуровневые абстракции в экосистеме Rust часто скрывают низкоуровневые детали работы с UNIX-сигналами и группами процессов, что при отправке сигналов (например, SIGINT / Ctrl+C) приводит к порче (некорректному сбросу флагов termios) управляющего терминала родительского процесса.
Для решения этой проблемы надо разработать библиотеку prux — **точный** порт подсистемы управления процессами из проекта tmux (C) на язык Rust (подсистема описана [здесь](./tmux-process-subsystem-architecture.md)). Логика tmux выбрана как эталон промышленной надежности в работе с PTY за более чем 15 лет эксплуатации.

## 2. Цели и задачи

* Вычленить из кодовой базы tmux исключительно подсистему управления процессами, исключив парсинг управляющих последовательностей терминала, отрисовку интерфейса и клиент-серверную архитектуру.
* Обеспечить строгую изоляцию дочерних процессов в собственных сессиях и группах процессов (Process Groups), исключающую утечку сигналов на родительский терминал.
* Предоставить неблокирующий I/O интерфейс для чтения и записи в PTY.
* Обеспечить кроссплатформенность в рамках POSIX-совместимых операционных систем (Linux, macOS, *BSD).

## 3. Этап 1: Разработка библиотеки prux (Порт подсистемы управления процессами tmux)
Библиотека должна представлять собой перевод на язык Rust логики файлов исходного кода tmux, с использованием низкоуровневых системных вызовов (крейты libc и/или nix)

## 3.1. Модуль prux::spawn (на базе spawn.c)

* Функционал: Создание пары PTY (Master/Slave) и порождение дочернего процесса.
* Критические требования (из tmux):
* Использование связки fork() и setsid() для создания новой сессии и отвязки от родительского терминала.
   * Назначение Slave-дескриптора управляющим терминалом нового процесса через ioctl(..., TIOCSCTTY, ...).
   * Настройка окружения и вызов dup2() для перенаправления стандартных потоков (0, 1, 2) на Slave PTY.
   * Финальный запуск целевого процесса через семейство вызовов exec.

## 3.2. Модуль prux::io (на базе window.c и bufferevent)

* Функционал: Чтение и запись данных в Master-дескриптор PTY.
* Критические требования:
* Перевод Master-дескриптора в неблокирующий режим (O_NONBLOCK).
   * Обеспечение возможности поллинга дескриптора.

## 3.3. Модуль prux::signal (на базе server.c и server-fn.c)

* Функционал: Обработка сигналов и мониторинг жизненного цикла процессов.
* Критические требования:
* Перехват SIGCHLD и вызов waitpid() в цикле с флагом WNOHANG для предотвращения образования процессов-зомби и "залипаний".
   * Использование паттерна "self-pipe" или аналогичного асинхронного пробуждения для безопасного проброса событий завершения процесса в основной цикл программы.

## 3.4. Модуль prux::jobs (на базе job.c)

* Функционал: Механизм запуска фоновых процессов без выделения полноценного PTY (для системных вызовов).

## 3.5. Модуль prux::os (на базе osdep-*.c и format.c)

* Функционал: Кроссплатформенное определение текущей директории (cwd) и PID выполняющегося в данный момент процесса внутри PTY.
* Реализация: Парсинг /proc (Linux), использование libproc (macOS) или системных вызовов sysctl (*BSD).

Ориентировочный суммарный объем кода библиотеки prux на языке Rust: 1500 – 2000 строк (LOC).

## 4. Проверка: Реализация интерфейса ProcessSession

Признаком готовности prux является возможность имплементации следующего интерфесйса (его имплементацию не нужно включать в prux, но нужно проверить что prux достаточен для его имплементации):

```
pub struct ProcessSessionConfig {
    pub program: PathBuf,
    pub args: Vec<String>,
    pub initial_cwd: PathBuf,
    pub debug_log_path: Option<PathBuf>,
}

pub enum ProcessError {
    Spawn(String),
    Io(String),
    Signal(String),
    CurrentDir(String),
    Unsupported(String),
}

pub struct ProcessSession;

impl ProcessSession {
    pub fn spawn(config: ProcessSessionConfig) -> Result<Self, ProcessError>;
    pub fn write_all(&mut self, bytes: &[u8]) -> Result<(), ProcessError>;
    pub fn try_read(&mut self) -> Result<Vec<u8>, ProcessError>;
    pub fn send_interrupt(&mut self) -> Result<(), ProcessError>;
    pub fn current_dir(&self) -> Result<PathBuf, ProcessError>;
    pub fn resize(&mut self, cols: u16, rows: u16) -> Result<(), ProcessError>;
    pub fn is_alive(&mut self) -> Result<bool, ProcessError>;
    pub fn terminate(&mut self) -> Result<(), ProcessError>;
}
```
